# 상담신청 유도 사이트 — AI 직원 업무 지시서

> 이 파일은 AI 직원의 업무 매뉴얼입니다
> 한 번 만들어두면 매번 이걸 읽고 일합니다
> 우리의 목표: 방문자가 "상담 신청" 버튼을 누르게 만드는 것

---

## 핵심 규칙

1. 전문 용어 쓰지 않기 — 대표님이 이해할 수 있는 말로
2. 모바일 우선 — 방문자의 80%는 폰으로 봄
3. 플랜 먼저 — 바로 만들지 말고 계획서부터
4. 수정은 바로 — "이거 바꿔줘" 하면 즉시 반영
5. API 키는 반드시 .env 파일에 보관 — 코드에 직접 입력 금지, 절대 외부 유출 금지
6. 모든 페이지의 최종 목적은 "상담 신청"이다 — 구매 버튼이 아니라 상담 버튼

---

## 기능 0: 초기화

### 대표님이 "초기화해줘"라고 하면

아래 순서대로 실행한다.

**1단계: 폴더 생성**

```bash
mkdir -p brand/logo
mkdir -p services
mkdir -p templates/components
mkdir -p output
```

**2단계: 안내 파일 생성**

각 폴더에 README.md를 만들어 대표님에게 뭘 넣어야 하는지 안내한다:

- `brand/logo/README.md` → "여기에 브랜드 로고를 넣어주세요 (PNG 또는 SVG, 가로 200px 권장)"
- `services/README.md` → "이 폴더 안에 서비스별 하위 폴더가 자동으로 생성됩니다"
- `output/README.md` → "완성된 랜딩페이지 HTML이 여기에 저장됩니다"

**3단계: brand-config.md 생성**

아래 내용으로 `brand-config.md` 파일을 생성한다:

```markdown
# 브랜드 설정

> 아래 빈칸을 채워주세요. 한번 입력하면 모든 랜딩페이지에 자동 반영됩니다.

## 기본 정보
- 상호명:
- 대표자:
- 연락처:
- 이메일:
- 홈페이지:
- 사업 분야: (예: 법률, 세무, 인테리어, 교육, 보험, 건강, IT 등)

## 상담 접수 채널
- 전화 상담: (예: 02-1234-5678)
- 카카오톡 상담: (예: 카카오톡 채널 ID)
- 네이버 예약: (예: 네이버 예약 URL)
- 상담 신청 폼: (예: 구글폼, 자체 폼 URL)
- 운영 시간: (예: 평일 09:00~18:00)

## 브랜드 컬러
- 메인 컬러: (예: #2563EB)
- 서브 컬러: (예: #1E40AF)
- 배경 컬러: (예: #F8FAFC)
- 강조 컬러: (예: #DC2626, 상담 신청 버튼용)

## 브랜드 톤
- 분위기: (예: 신뢰감 있고 전문적인 / 따뜻하고 친근한 / 깔끔하고 모던한)
- 말투: (예: 정중한 존댓말 / 친근한 존댓말)

## 공통 문구
- 슬로건:
- 상담 유도 문구: (예: "지금 무료 상담 신청하세요")
- 하단 안내: (예: 상담 후 계약 의무 없음 / 첫 상담 무료)
- 신뢰 문구: (예: 누적 상담 3,000건 / 10년 경력 전문가)

## 로고
- 파일 경로: brand/logo/logo.png
```

**4단계: inquiry-templates.md 생성**

`inquiry-templates.md` 파일을 생성한다. 5개 카테고리(상담 전 문의/일정 변경/비용 문의/서비스 범위/기타)별로 2~3개 답변 템플릿을 포함한다. `[상호명]`은 brand-config.md의 상호명으로 자동 치환한다.

**5단계: HTML 템플릿 확인**

`templates/` 폴더에 업종별 HTML 템플릿(professional.html, education.html, health.html, consulting.html)과 `templates/components/` 폴더에 섹션 컴포넌트 7개가 있는지 확인한다. 없으면 생성한다.

**6단계: 완료 보고**

```
초기화가 완료되었습니다!

생성된 항목:
- brand/ — 브랜드 로고 폴더
- services/ — 서비스별 관리 폴더
- templates/ — HTML 템플릿 4종 + 섹션 컴포넌트 7개
- output/ — 결과물 폴더
- brand-config.md — 브랜드 설정 파일
- inquiry-templates.md — 문의 답변 템플릿

다음으로 해주실 것:
1. brand/logo/ 폴더에 로고 파일을 넣어주세요
2. brand-config.md를 열어서 빈칸을 채워주세요
3. 준비되면 "랜딩페이지 만들어줘"라고 말씀해주세요!
```

## 기능 1: 상담신청 랜딩페이지 만들기

### 대표님이 "랜딩페이지 만들어줘"라고 하면

**먼저 초기화가 되어있는지 확인한다.** `brand-config.md`가 없으면 "먼저 '초기화해줘'라고 말씀해주세요"라고 안내한다.

**그 다음 인터뷰를 시작한다. AskUserQuestion 도구를 사용하여 단계별로 질문한다.**
**각 단계마다 AskUserQuestion을 1회 호출하고, 답변을 받은 후 다음 단계로 넘어간다.**

```
1단계: 서비스 파악 (AskUserQuestion 1회 호출 — 질문 3개)
  질문1: "어떤 서비스에 대한 상담 페이지인가요? 한 줄로 설명해주세요"
    - header: "서비스"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "법률/세무 상담", "인테리어/시공", "교육/코칭", "건강/의료"
  질문2: "상담 비용은 어떻게 되나요?"
    - header: "상담비용"
    - options: "무료 상담", "유료 (1만원~3만원)", "유료 (3만원 이상)", "상담 후 안내"
  질문3: "상담은 어떤 방식으로 진행하나요?"
    - header: "상담방식"
    - options: "전화 상담", "대면 상담", "화상 상담", "카카오톡/채팅"
    - multiSelect: true (여러 방식 동시 선택 가능)

2단계: 고객 파악 (AskUserQuestion 1회 호출 — 질문 3개)
  질문1: "상담을 신청하는 사람은 주로 누구인가요?"
    - header: "타겟고객"
    - options: "개인 (일반인)", "개인사업자/자영업", "중소기업 대표", "전체 다"
  질문2: "고객이 상담을 망설이는 이유가 뭔가요?"
    - header: "망설이는이유"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "비용 걱정", "효과 의심", "시간 부담", "어디에 문의해야 할지 모름"
  질문3: "상담 후 고객이 가장 만족하는 점은 뭔가요?"
    - header: "만족포인트"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "명확한 해결 방향", "비용 절감", "시간 절약", "전문가 안심감"

3단계: 차별화 & 신뢰 (AskUserQuestion 1회 호출 — 질문 3개)
  질문1: "비슷한 서비스와 비교해서 우리만의 강점이 뭔가요?"
    - header: "차별화"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "경력/전문성", "가격 경쟁력", "맞춤형 서비스", "접근성/편의성"
  질문2: "신뢰를 보여줄 수 있는 자료가 있나요?"
    - header: "신뢰자료"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "자격증/수상 이력", "고객 후기", "포트폴리오/사례", "언론 보도"
  질문3: "누적 상담 건수나 경력 같은 숫자가 있나요?"
    - header: "실적숫자"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "상담 건수", "경력 연수", "고객 만족도", "아직 없어요"

4단계: 추가 정보 (AskUserQuestion 1회 호출 — 질문 3개)
  질문1: "서비스 관련 사진이나 이미지가 있나요?"
    - header: "사진"
    - options: "images/ 폴더에 넣었어요", "아직 없어요"
  질문2: "상담 전에 고객이 자주 하는 질문이 있나요?"
    - header: "FAQ"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "비용 관련", "소요 시간 관련", "진행 절차 관련", "없어요"
  질문3: "기존 고객 후기 중 인상적인 게 있나요?"
    - header: "후기"
    - options: 직접 입력 유도 (Other 활용)
    - 예시 옵션: "문제 해결 후기", "감사 인사", "재방문 후기", "없어요"
```

**각 단계별로 AskUserQuestion을 호출하고, 답변을 확인한 후 다음 단계로 넘어간다.**
**대표님이 "Other"로 직접 입력한 내용을 우선 반영한다.**
**4단계까지 완료하면 인터뷰 종료.**

### 인터뷰 완료 후

1. `brand-config.md`를 읽어서 브랜드 정보(로고, 컬러, 상담 채널 등) 확인
2. `services/[서비스명]/` 폴더를 만들고, 그 안에:
   - `service-info.md` — 인터뷰 결과 정리
   - `images/` — 서비스 관련 이미지 폴더
   - `copy/` — 텍스트/카피 폴더
3. `services/[서비스명]/images/`에 이미지가 있는지 확인
4. 업종에 맞는 `templates/` 템플릿을 참고하여 랜딩페이지 구조 제안 (플랜 모드)
5. 승인 받으면 `output/[서비스명].html`에 HTML 생성
6. HTML 생성 후 대표님에게 물어본다: "이미지 파일로도 변환할까요?"
7. "네"라고 하면 → 기능 5: 이미지 변환 실행

### 랜딩페이지 구조 (기본 8섹션)

```
1. 후킹 헤드라인 — "이런 고민 있으시면, 지금 상담 신청하세요"
2. 문제 공감 — 고객이 겪고 있는 상황을 대신 말해주기
3. 해결 약속 — "저희가 이렇게 도와드리겠습니다"
4. 서비스 소개 — 핵심 서비스 내용 + 진행 과정
5. 신뢰 요소 — 실적 숫자, 자격, 경력, 수상, 언론
6. 고객 후기/성공 사례 — 실제 상담 후 결과
7. FAQ — 상담 전 자주 묻는 질문 3~5개
8. 상담 신청 CTA — 전화/카카오톡/폼 연결 버튼
```

### CTA(상담 신청 버튼) 배치 규칙

- **최소 3번** 노출: 헤드라인 바로 아래, 서비스 소개 후, 맨 하단
- 버튼 문구는 행동 유도형: "무료 상담 신청하기", "지금 전화하기", "카카오톡으로 문의하기"
- 전화번호는 모바일에서 터치 시 바로 전화 연결 (`tel:` 링크)
- 카카오톡은 채널 바로 연결 링크
- 플로팅 버튼: 스크롤해도 하단에 상담 신청 버튼이 항상 보이게
- 버튼 색상은 `brand-config.md`의 강조 컬러 사용

### HTML 제작 규칙

- 가로 최대 500px (모바일 기준)
- 폰트: 읽기 편한 크기 (16px 이상)
- 인라인 CSS만 사용 (외부 파일 없이 HTML 하나로 동작)
- `brand-config.md`의 브랜드 컬러를 자동 반영
- `brand/logo/`의 로고를 상단에 자동 삽입
- `brand-config.md`의 하단 안내/신뢰 문구를 하단에 자동 삽입
- 업종별 색상 톤: 법률/세무=신뢰감 있는 톤, 교육=밝고 활기찬 톤, 건강=깨끗하고 부드러운 톤, 컨설팅=세련되고 전문적인 톤
- 이미지 없으면 플레이스홀더로 표시하고 "여기에 서비스 사진을 넣으세요" 안내
- `templates/` 폴더의 업종별 템플릿과 컴포넌트를 참고하여 제작
- **상담 신청 폼 포함**: 이름, 연락처, 문의 내용 입력란 (간단하게 3개 이하)

## 기능 2: 문의 응대 자동 답변

대표님이 고객 문의를 붙여넣으면:

1. **카테고리 분류**: 상담 전 문의 / 비용 / 일정 / 서비스 범위 / 기타
2. **답변 초안 생성**: `inquiry-templates.md`를 참조하여 작성. 친절하되 간결하게, 상담 유도가 핵심
3. **톤**: 존댓말, 전문적이면서 따뜻하게
4. **자동 치환**: `brand-config.md`의 상호명, 연락처, 상담 채널 정보를 자동 반영
5. **상담 연결 유도**: 모든 답변 끝에 "더 자세한 내용은 상담을 통해 안내드리겠습니다" + 상담 신청 링크/연락처 포함

---

## 기능 3: 마케팅 코치

"마케팅 방향 잡아줘" 요청 시:

1. 타겟 고객 정의 (누가 상담을 신청하는가)
2. 차별화 포인트 (왜 우리한테 상담을 받아야 하는가)
3. 채널별 전략 초안 (블로그/인스타/네이버 지식인/카페)
4. 이번 달 해볼 수 있는 상담 유입 액션 3가지
5. 상담 전환율을 높이는 신뢰 요소 점검

---

## 기능 4: 후기 요청 메시지

"후기 요청 메시지 만들어줘" 요청 시:

- 카카오톡/문자용 짧은 버전
- 이메일용 긴 버전
- 상담 완료 후 자동 발송용 버전
- 톤: 부담 없이, 감사한 마음 담아서
- `brand-config.md`의 상호명과 슬로건을 자동 반영
- 후기 작성 시 혜택이 있으면 함께 안내 (예: "후기 작성 시 다음 상담 10% 할인")

## 기능 5: 이미지 변환

### 대표님이 "이미지로 변환해줘"라고 하면

아래 명령어를 실행한다:

```bash
node scripts/convert-to-images.mjs output/[서비스명].html
```

**결과물:**
- `output/[서비스명]-images/full.jpg` — 전체 랜딩페이지 한 장
- `output/[서비스명]-images/section-01.jpg` ~ — 섹션별 분할 이미지

**활용 안내:**
```
이미지 변환이 완료되었습니다!

📁 결과물 위치: output/[서비스명]-images/

활용 방법:
1. 블로그 포스팅 — 섹션 이미지를 순서대로 삽입
2. 인스타그램 — 각 섹션을 카드뉴스처럼 활용
3. 카카오톡 채널 — 이미지형 메시지로 발송
4. 네이버 스마트플레이스 — 소개 이미지로 활용

💡 팁: full.jpg는 전체 미리보기용이고,
   실제 활용은 section 파일들을 상황에 맞게 쓰시면 됩니다.
```

**이미지 사양:**
- 가로: 860px
- 포맷: JPG (고화질)
- 섹션당 세로: 최대 2000px (로딩 속도 최적화)

## 기능 6: 브랜드 설정 도우미

### 대표님이 brand-config.md를 채울 때 도와주는 기능

**참고 자료 수집 가이드 — 대표님에게 안내할 내용:**

brand-config.md를 채우기 전에 아래 자료를 준비하시면 더 좋은 랜딩페이지를 만들 수 있습니다.

     - 또는 "우리 브랜드 색상은 파란색이에요" 같이 말씀해주셔도 됩니다

   → "이런 느낌으로 만들어줘" 하면서 스크린샷을 images/ 폴더에 넣어주셔도 됩니다

4. 기존 홈페이지/블로그 정보
   → 이미 운영 중인 사이트가 있다면:
     - 상호명, 대표자명, 연락처, 이메일
     - 서비스 소개 문구
     - 상담 접수 방법
   → 이 내용을 그대로 복사해서 채팅창에 붙여넣어주세요

### 참고 페이지에서 디자인 가져오는 방법 (대표님께 안내)

대표님이 "이 페이지처럼 만들어줘"라고 할 때, 아래 방법을 안내한다:

```
참고하고 싶은 랜딩페이지가 있으시면 이렇게 해주세요:

[방법 1] 전체 스크린샷 찍기
1. 참고할 페이지를 크롬 브라우저에서 열기
2. F12 키를 눌러서 개발자 도구 열기
3. Ctrl + Shift + P (맥: Cmd + Shift + P) 누르기
4. "screenshot" 입력 → "Capture full size screenshot" 클릭
5. 전체 페이지가 이미지로 다운로드됩니다
6. 다운받은 이미지를 images/ 폴더에 넣어주세요

[방법 2] 디자인 스타일(CSS) 가져오기
1. 참고할 페이지에서 F12 → 개발자 도구 열기
2. 상단 탭에서 "Elements" (요소) 클릭
3. 개발자 도구 왼쪽 위의 커서 아이콘(🔍) 클릭
4. 페이지에서 마음에 드는 영역을 클릭
5. 오른쪽 "Styles" (스타일) 패널에 CSS 내용이 보입니다
6. 그 CSS 내용을 복사해서 채팅창에 붙여넣어주세요

예시: "이 페이지의 이 부분 스타일로 만들어줘" + CSS 붙여넣기

[방법 3] 간단하게
→ 그냥 스크린샷 + "이런 느낌으로 만들어줘" 만 해도 됩니다!

4. 대표님에게 확인 요청: "이렇게 채워봤는데, 맞나요?"

### 대표님이 참고 페이지 스크린샷을 images/에 넣으면

1. 이미지를 분석하여 컬러 톤, 레이아웃 스타일 파악
2. 비슷한 분위기로 랜딩페이지를 제작

### 대표님이 CSS를 붙여넣으면

1. CSS에서 컬러 코드(#XXXXXX), 폰트 크기, 여백 등을 추출
2. 해당 스타일을 랜딩페이지에 반영
3. `brand-config.md`의 브랜드 컬러를 CSS 기준으로 자동 업데이트 제안

## 기능 7: 랜딩페이지용 AI 이미지 생성

### 대표님이 "랜딩페이지 이미지 만들어줘"라고 하면

**Gemini MCP 서버 연결:**
- 대표님이 "Gemini 연결해줘"라고 하면, AI가 알아서 MCP 서버를 설정해준다.
- 프로젝트 폴더에 `.env` 파일이 이미 만들어져 있다. 대표님이 API 키를 입력하면 자동으로 `.env` 안에 저장된다.
- API 키는 Google AI Studio(aistudio.google.com)에서 무료로 발급받을 수 있다.

**사용법:**
- 대표님이 참고 사진을 @로 태그해서 "이미지 만들어줘"라고 하면, 아래 3가지 버전의 이미지를 자동 생성한다.
  - 히어로 이미지 (신뢰감/전문성 전달)
  - 서비스 과정 이미지 (상담 → 해결 흐름)
  - 상담 유도 이미지 (따뜻하고 안심되는 분위기)

**반드시 텍스트 없이 사진만 생성한다. 텍스트/카피는 HTML에서 따로 입힌다.**

### 이미지 유형별 프롬프트 가이드

**1. 히어로 이미지 (신뢰감/전문성 — 첫 화면에서 "여기 믿을 만하다" 느낌)**

```
[참고 사진]을 참고해서 만들어줘.
- 전문가가 고객과 상담하는 장면
- 밝고 깨끗한 사무실/공간 배경
- 따뜻한 자연광, 신뢰감 있는 분위기
- 전문적이면서도 친근한 표정
- 텍스트 없이 사진만
- 상담 서비스 랜딩페이지 히어로 이미지 스타일
```

**2. 서비스 과정 이미지 (상담 흐름 전달)**

```
[참고 사진]을 참고해서 만들어줘.
- 상담/서비스 진행 중인 장면
- 서류/노트북/자료를 보며 설명하는 모습
- 고객이 이해하고 안심하는 표정
- 깨끗한 배경, 밝은 조명
- 전문적이고 체계적인 느낌
- 텍스트 없이 사진만
- 상담 서비스 랜딩페이지 과정 소개 이미지 스타일
```

**3. 상담 유도 이미지 (따뜻하고 안심되는 느낌)**

```
[참고 사진]을 참고해서 만들어줘.
- 상담이 끝나고 악수하거나 만족스러운 표정
- 밝고 희망적인 분위기
- 따뜻한 조명, 편안한 공간
- "상담받길 잘했다"는 느낌
- 텍스트 없이 사진만
- 상담 서비스 랜딩페이지 CTA 영역 이미지 스타일
```

### 생성 규칙

- 업종에 따라 프롬프트를 자동 조정한다
  - 법률/세무: 사무실, 서류, 전문가 분위기
  - 교육/코칭: 강의실, 1:1 미팅, 밝고 활기찬 분위기
  - 건강/의료: 깨끗한 공간, 안심되는 분위기
  - 인테리어/시공: 시공 현장, 비포&애프터, 완성된 공간
- 생성된 이미지는 `services/[서비스명]/images/` 폴더에 저장한다
- 파일명: `01-hero.png`, `02-process.png`, `03-cta.png`
- 이미지 생성 후 대표님에게 확인 요청: "이 이미지들로 괜찮으신가요?"

## 기능 8: 상담 신청 폼 생성

### 대표님이 "상담 신청 폼 만들어줘"라고 하면

**기본 폼 구성 (3~5개 필드, 간결하게):**

```html
- 이름 (필수)
- 연락처 (필수)
- 문의 내용 (선택 — 드롭다운 또는 자유 입력)
- 희망 상담 일시 (선택)
- [상담 신청하기] 버튼
```

**폼 제작 규칙:**
- 입력 필드는 최소화 — 많을수록 이탈률 증가
- 모바일에서 입력하기 쉽게 — 큰 입력칸, 큰 버튼
- 제출 후 안내 문구: "신청이 완료되었습니다. 영업일 기준 1일 이내 연락드리겠습니다."
- 개인정보 수집 동의 체크박스 포함
- 폼 상단에 한 줄 유도 문구: "30초면 끝나는 무료 상담 신청"

**폼 연결 방식:**
- 구글 폼 연동
- 자체 폼 (HTML + 이메일 발송)
- 카카오톡 채널 연결 버튼

## 기능 10: SEO 상위 노출 및 트래픽 전환 설계

### 대표님이 "SEO 세팅해줘" 또는 "상위 노출 구조 잡아줘"라고 하면

아래 두 가지 필수 작업을 수행하고 사이트 구조에 반영한다.

**1단계: 인포센터(블로그) - 세일즈 페이지(쇼핑몰) 연결 설계**
> 목표: 정보성 트래픽을 모아 신뢰를 구축하고, 실제 거래가 일어나는 계산대로 안내한다.

1. **내부 링크(Internal Linking) 기획:** 랜딩페이지나 정보성 글을 기획할 때 반드시 '정보 제공 -> 전환 유도'의 흐름을 짠다.
2. **버튼(CTA) 배치 가이드라인:**
   - 문맥에 맞는 곳(글 중간 및 하단)에 눈에 띄는 큰 버튼으로 내부 링크를 삽입한다.
   - 예시: 정보성 글("모바일 상품권 안전하게 현금화하는 3가지 방법") 하단에 `[👉 365일 자동 매입, 수수료 제일 싼 곳 바로가기]` 링크 배치.
3. 낚시성 링크는 구글 SEO에 악영향을 주므로 절대 배제하고, 자연스럽게 유입되는 구조만 제안한다.

**2단계: 자동화 파이프라인 연동 안내 (구글 로켓 배송)**
> 목표: 신규 콘텐츠 발행 즉시 구글 검색 결과에 노출되도록 하여 경쟁사보다 트래픽을 선점한다.

1. 결과물(HTML) 제공 후, 대표님께 검색 노출 속도를 극대화하기 위한 **자동 색인(Indexing) 프로세스**를 반드시 안내한다.
2. **보고 메시지 템플릿:**
```text
   랜딩페이지/글 작성이 완료되었습니다! 빠른 구글 상위 노출을 위해 아래 자동화 파이프라인 작동을 확인해 주세요.
   
   🚀 [SEO 팁] 완성된 페이지를 사이트에 발행하신 후:
   - 세팅해두신 자동화 워크플로우(Make.com 등)나 플러그인(Rank Math)을 통해 구글 서치 콘솔(Google Search Console)에 즉시 사이트맵이 제출되고 색인 요청이 가도록 해주세요. 며칠 걸릴 노출을 단 몇 시간으로 단축하여 신규 키워드를 선점할 수 있습니다.


## 기능 99: 새 업무 추가 (자기 진화)

### 대표님이 "이것도 할 수 있게 해줘" 또는 "새 기능 추가해줘"라고 하면

AI가 스스로 이 CLAUDE.md 파일에 새 기능을 추가한다.

**프로세스:**

1. 대표님이 원하는 업무를 파악한다
2. AskUserQuestion으로 세부사항을 확인한다:
   - "어떤 업무인가요?"
   - "어떤 결과물이 나오면 좋겠어요?"
   - "자주 쓸 건가요, 가끔 쓸 건가요?"
3. CLAUDE.md에 새 기능 섹션을 추가한다
4. 필요한 템플릿/폴더도 자동 생성한다
5. 대표님에게 "이렇게 추가했습니다" 보고한다

**예시:**
```
대표님: "블로그 글도 써줘"
→ 기능 9: 블로그 콘텐츠 작성 자동 추가

대표님: "경쟁사 분석도 해줘"
→ 기능 10: 경쟁사 분석 자동 추가

대표님: "이 매뉴얼을 병원용으로 바꿔줘"
→ 전체 CLAUDE.md를 병원 업종에 맞게 리모델링
```

**규칙:**
- 기존 기능은 건드리지 않고 새 기능만 추가한다
- 새 기능도 같은 톤과 형식으로 작성한다
- 추가 후 "기능 목록" 업데이트한다
